What is a Data Center?

Definition

A data center is a facility that houses computing infrastructure - servers, storage systems, and networking equipment - designed to process, store, and distribute data.

Key Components:
  • Servers - Compute workloads
  • Storage - SAN, NAS, object storage
  • Network - Switches, routers, firewalls
  • Power - Redundant power, UPS, generators
  • Cooling - HVAC, hot/cold aisles

Data Center Types

Enterprise Data Center
Owned/operated by single organization
Colocation (Colo)
Rent space in shared facility
Cloud Provider
AWS, Azure, GCP data centers

Real-World Analogy

A data center is like a warehouse for computers. Just as a warehouse needs loading docks (network connections), shelving (racks), climate control, and security - a data center needs similar infrastructure for its digital inventory.

Three-Tier Architecture

The traditional data center network design uses three hierarchical layers:

🔹 Core Layer High-speed packet switching | Interconnects everything
🔸 Distribution/Aggregation Layer Routing | Filtering | QoS | Security policies
🔷 Access/Edge Layer Server & device connectivity | VLANs | Port security

Why Three Tiers?

  • Scalability: Add devices at each layer independently
  • Redundancy: Multiple paths between layers
  • Manageability: Clear separation of functions
  • Performance: Optimized for each layer's role
Slide 3 of 18 Network Essentials

Core Layer

Core Layer The "backbone" of the network

Primary Functions

  • High-speed switching - Fast packet forwarding
  • Interconnectivity - Connects all network pods
  • Reliability - No single point of failure
  • Low latency - Minimal processing delay
Important: The core layer should NOT perform packet manipulation (filtering, NAT, ACLs). Its only job is FAST forwarding!

Core Layer Characteristics

Speed: Highest bandwidth (40G, 100G, 400G links)
Redundancy: Fully meshed connections
Simplicity: Minimal configuration

Typical Devices

  • High-performance Layer 3 switches
  • Core routers (Cisco Nexus, Arista, Juniper)
  • 40G/100G SFP+ and QSFP+ modules

Distribution/Aggregation Layer

Distribution Layer The "intelligent" layer

Primary Functions

  • Aggregation - Combines access layer uplinks
  • Routing - Inter-VLAN routing
  • Policy enforcement - ACLs, QoS
  • Security - Firewall integration
  • Redundancy - STP, HSRP/VRRP

This Layer Handles

Access Control Lists (ACLs)
Filter traffic based on rules
Quality of Service (QoS)
Prioritize critical traffic
Route Summarization
Reduce routing table size to core
Load Balancing
Distribute traffic across paths

Analogy

Think of the distribution layer as airport security. It checks IDs (ACLs), prioritizes VIP passengers (QoS), and directs traffic to the right gates (routing). The core layer is just the runway - fast and simple!

🖥 Server A
←→
🖥 Server B
Definition: Traffic between devices INSIDE the data center
  • App server to database server
  • VM to VM communication
  • Storage replication

Modern Data Centers

Today, East-West traffic dominates (up to 80%+) due to distributed applications, microservices, and virtualization. This is why the Spine-Leaf architecture is becoming more popular - it's optimized for East-West traffic!

Spine-Leaf: Why It's Better for Modern DC

📈

Predictable Latency

Every server is exactly 2 hops from every other server. No matter which path, same latency!

🔧

Easy Scaling

Add more leaf switches for more servers. Add more spine switches for more bandwidth. Simple!

⚡

High Bandwidth

Multiple paths between any two points. Uses ECMP (Equal-Cost Multi-Path) for load balancing.

🔒

No STP Needed

Layer 3 everywhere (routed links) - no Spanning Tree blocking. All links are active!

Aspect Three-Tier Spine-Leaf
Hop Count Variable (3-5 hops) Consistent (2 hops)
East-West Traffic Suboptimal (hairpinning) Optimized
Scaling Vertical (bigger switches) Horizontal (more switches)
STP Dependency Yes (blocks links) No (all links active)

Software-Defined Networking (SDN)

What is SDN?

SDN separates the control plane (decision making) from the data plane (packet forwarding), centralizing network intelligence.

Application Layer Apps & Services
Control Layer SDN Controller
Infrastructure Layer Network Devices

SDN Benefits

Centralized Management
Manage entire network from one controller
Programmability
Automate via APIs, no manual CLI
Agility
Deploy changes in minutes, not days
Vendor Neutral
Mix hardware from different vendors

Analogy

Traditional networking is like each intersection having its own traffic cop making independent decisions. SDN is like having a central traffic control center that sees ALL intersections and coordinates them intelligently!

SDN: Control, Data, and Management Planes

The Three Planes

Management Plane
Configure and monitor
SSH, SNMP, APIs, GUI dashboards
How admins interact with devices
Control Plane
Make decisions
Routing protocols (OSPF, BGP)
Build routing/forwarding tables
Data Plane (Forwarding Plane)
Move packets
Actual forwarding based on tables
Hardware-level switching

Traditional vs SDN

Traditional

Control + Data planes are together in each device. Every switch/router makes its own decisions.

SDN

Control plane is centralized in the SDN controller. Devices just forward packets based on instructions.

Key Protocol: OpenFlow is the standard protocol for communication between SDN controller and network devices.

Virtualization in Data Centers

Server Virtualization

Run multiple Virtual Machines (VMs) on a single physical server using a hypervisor.

Hypervisor Types:
  • Type 1 (Bare Metal): VMware ESXi, Hyper-V, KVM
  • Type 2 (Hosted): VirtualBox, VMware Workstation

Benefits

  • Better hardware utilization
  • Isolation between workloads
  • Easy backup and migration
  • Rapid provisioning

Network Virtualization

Create virtual networks that are decoupled from physical hardware.

Key Technologies:
  • VLANs: Segment physical network
  • VXLAN: Extend L2 over L3 (overlay)
  • Virtual Switches: vSwitch, Open vSwitch
  • NFV: Network Functions Virtualization
NFV: Run network functions (firewall, load balancer, IDS) as software instead of dedicated hardware appliances.

Redundancy & High Availability

Power Redundancy

N+1 Redundancy:
One extra power supply beyond requirement
2N Redundancy:
Fully duplicate power systems (A+B feeds)
Components:
  • UPS (Uninterruptible Power Supply)
  • Diesel generators
  • Dual power feeds
  • PDUs (Power Distribution Units)

Network Redundancy

Techniques:
  • Dual-homed servers: Two NICs
  • Link aggregation: Bonded links
  • Multiple uplinks: Redundant paths
  • HSRP/VRRP: Gateway redundancy
  • Diverse paths: Different physical routes

Physical Design: Hot Aisle / Cold Aisle

The Problem

Servers generate massive heat. Without proper cooling, equipment overheats and fails.

The Solution: Hot/Cold Aisles

Arrange racks so cold air intakes face each other (cold aisle) and hot air exhausts face each other (hot aisle).

How It Works:
  1. CRAC units blow cold air into cold aisle
  2. Servers intake cold air from front
  3. Hot air exhausts out the back
  4. Hot air rises and returns to cooling

Containment: Modern data centers use physical barriers (doors, curtains) to prevent hot and cold air from mixing.

Data Center Storage

🖧

DAS (Direct Attached Storage)

Storage directly connected to a server. Simple but not shared.

Example: Internal hard drives, USB drives

🌐

NAS (Network Attached Storage)

File-level storage accessed over the network (TCP/IP). Easy to manage.

Protocols: NFS, SMB/CIFS

⚡

SAN (Storage Area Network)

Block-level storage over dedicated high-speed network. High performance.

Protocols: Fibre Channel, iSCSI, FCoE

☁

Object Storage

Stores data as objects with metadata. Highly scalable for cloud.

Examples: AWS S3, Azure Blob

Type Access Method Best For
DAS Direct connection Single server, low cost
NAS File-level (NFS/SMB) File sharing, backups
SAN Block-level (FC/iSCSI) Databases, VMs
Object HTTP/REST API Cloud apps, big data

Key Exam Points

Must Know for Network+

Q: What are the three layers of three-tier architecture?

A: Core, Distribution/Aggregation, Access

Q: What is North-South traffic?

A: Traffic entering/leaving the data center

Q: What is East-West traffic?

A: Traffic between servers inside the data center

Architecture Comparison

Q: Why is Spine-Leaf better for East-West?

A: Predictable 2-hop latency, all links active (no STP), easy horizontal scaling

Q: What does SDN centralize?

A: The control plane (decision making)

Q: Which layer does NOT filter traffic?

A: Core layer (only fast forwarding)

Memory Aid: Think of the three tiers as:
Core = Highway (fast, no stops) | Distribution = Checkpoint (security, routing) | Access = Parking lot (where you connect)

Key Takeaways

  • Three-Tier: Traditional, hierarchical (Core → Distribution → Access)
  • Spine-Leaf: Modern, flat, optimized for East-West traffic
  • Traffic Flows: North-South (in/out) vs East-West (internal)
  • SDN: Centralizes control plane, enables automation
  • Virtualization: VMs, virtual networks, NFV
  • Redundancy: Power (N+1, 2N), network (multiple paths), cooling
